# LOL 战队数据看板

这是一个基于 Python 与 Streamlit 构建的本地电竞数据分析小应用。每次更新 Excel 后，重新上传文件即可无缝刷新多维度的战队数据看板与对战预测。

## 功能特性

### 1. 单队深度数据看板
- **多维度统计**：提供整体、蓝方、红方分类下的胜率、平均比赛时长统计。
- **IQR 异常值剔除**：当某战队已完成比赛达到或超过 6 场时，自动通过四分位距（IQR）法识别并剔除比赛时长的异常值，确保平均数不被极端局失真；未参与平均计算的对局会在明细中清晰标注。
- **前期节奏分析**：直观展示战队在拿到**【一血】**和**【一塔】**时的胜率转化关系。
- **赛程管理**：支持已完赛对局的详细指标追踪，并可通过 `team1` 与 `team2` 字段自动加载尚未开打的待赛赛程。

### 2. 赛事对战预测系统
- **H2H 动态权重混合**：选择两支战队与首局蓝方后，系统会自动检索双方的历史交锋记录（H2H）。若存在交手历史，将动态引入 H2H 权重与常规数据加权混合计算，提供更贴近实战的预测。
- **单局对阵细化**：逐局预测胜率、比赛时长合理区间、一血率、一塔率以及各项资源（击杀数、推塔数、小龙数、大龙数）的预期表现。
- **系列赛分列概率预测**：提供 BO1、BO3、BO5 全方位的系列赛胜率及比分概率。比分结果采用**分行分列的对称网格结构**展示（如 3:0 对应 0:3），并自动将**每一组赛制中概率最高的结果标绿加粗**，大幅提升盘口与赛果的可读性。

## 首次安装

在此文件夹打开终端，执行依赖安装：

```bash
python3 -m pip install -r requirements.txt
```

## 启动

```bash
python3 -m streamlit run app.py
```

浏览器会自动打开。点击“选择比赛数据 Excel”，选择你的 `LOL Esport.xlsx`。

## Excel 数据要求

工作表名称必须为 `match`，并至少包含以下列：

- blue：蓝色方战队名称
- red：红色方战队名称
- win：本局获胜战队名称（留空则判定为尚未进行的待赛赛程）
- duration：比赛时长（支持标准时间文本格式）
- first_kill：一血获得战队名称
- first_tower：一塔获得战队名称
- kill_blue / kill_red：蓝方 / 红方击杀数
- tower_blue / tower_red：蓝方 / 红方推塔数
- dragon_blue / dragon_red：蓝方 / 红方小龙数
- nashor_blue / nashor_red：蓝方 / 红方大龙数

注：若 Excel 中包含 team1 与 team2 列，应用将自动识别并拓展展示未来的待赛赛程表。
